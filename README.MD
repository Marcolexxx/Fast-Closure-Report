## рҹ“– Overview

**The problem:** After every large-scale marketing event, execution teams spend 4вҖ“8 hours manually cross-checking materials lists, sorting on-site photos, reconciling payment vouchers, and hammering everything into a closure report deck. The process is multi-person, multi-format, and painfully error-prone вҖ” mismatched quantities and unreconciled invoices are the norm, not the exception.

**The solution:** Fast Closure Report ships a 12-step agentic pipeline. The system auto-parses Excel BOMs, pulls photos from cloud albums, runs visual AI detection, and pauses at three structured human decision points вҖ” an annotation canvas, a design-binding UI, and a receipt reconciliation panel вҖ” before auto-resuming and producing a reviewer-ready PPTX closure report.

**What it gives you:**
- **Skill-as-Plugin architecture** вҖ” new business workflows ship as `skill.json` + `tools/*.py`. Zero changes to platform core.
- **Step-level checkpointing** вҖ” tasks resume from their last checkpoint after any interruption.
- **Experience Layer** вҖ” every HIL correction is distilled into searchable `LibrarianKnowledge`, continuously improving AI accuracy over time.
- **Full observability** вҖ” Trace-IDs flow from HTTP вҶ’ Celery вҶ’ WebSocket; every tool invocation is logged and auditable.

## вңЁ Killer Features

- рҹӨ– **12-step automated pipeline** вҖ” `parse_excel вҶ’ AI detection вҶ’ OCR вҶ’ receipt matching вҶ’ PPT generation` вҖ” triggered with a single API call
- рҹ§‘вҖҚрҹ’» **Human-in-the-Loop (HIL)** вҖ” structured human intervention at annotation, design binding, and receipt confirmation вҖ” with pre-filled AI suggestions
- рҹ”Ң **Skill-as-Plugin** вҖ” each business domain is an isolated Skill: JSON declaration + Python tools, hot-reloadable at runtime
- рҹҺ“ **Experience Layer** вҖ” HIL corrections вҶ’ `FeedbackEvent` вҶ’ `LibrarianKnowledge` (PostgreSQL TSVECTOR full-text search) вҖ” the system gets smarter with every run
- рҹ”Қ **PatternMiner** вҖ” Celery Beat scans вүҘ50 correction records on a schedule and generates high-frequency error reports with Prompt-tuning recommendations
- вҡЎ **Async AI detection** вҖ” heavy vision inference is offloaded to a Celery Worker + a dedicated inference service; the ASGI worker thread is never blocked
- рҹ”ҙ **Real-time WebSocket progress** вҖ” Redis Pub/Sub drives millisecond-level step and HIL state updates to the frontend
- рҹӣЎпёҸ **DLQ Guardian** вҖ” `task_guardian_patrol` auto-fails tasks stuck in `RUNNING` for over an hour, preventing pipeline deadlocks
- рҹ—‘пёҸ **Soft-delete + 7-day recycle bin** вҖ” `AssetImage` and `ProjectFile` support logical deletion; a GC Beat task handles physical cleanup
- рҹ”җ **RBAC with 4 roles** вҖ” `executor / reviewer / finance / admin`; WebSocket layer enforces IDOR protection

## рҹҡ§ Troubleshooting

### вқҢ Issue 1: Frontend gets 401 Unauthorized after API restart

**Root cause:** `SECRET_KEY` is not set to a fixed value in `.env`. On every container start, `config.py:16` calls `secrets.token_hex(32)`, generating a new random key and instantly invalidating all issued JWTs.

**Fix:**
```bash
python -c "import secrets; print(secrets.token_hex(32))"
# Paste the output into .env: SECRET_KEY=<output>
docker compose restart api worker
```

---

### вқҢ Issue 2: `docker compose up --build` fails вҖ” pip install timeout

**Root cause:** `backend/Dockerfile` hardcodes Tsinghua University's PyPI mirror (`pypi.tuna.tsinghua.edu.cn`). This mirror is unreachable outside mainland China.

**Fix:** Edit `backend/Dockerfile` lines 5вҖ“6 and remove the mirror configuration:
```dockerfile
RUN pip install --no-cache-dir --default-timeout=120 --retries=10 -r requirements.txt
```
Alternatively, set `PIP_INDEX_URL` to a mirror that works in your region.

---

### вқҢ Issue 3: AI detection tasks get stuck in RUNNING, then flip to ERROR

**Root cause A:** `run_async()` in `celery_tasks.py` spawns one thread per Celery task to bridge async SQLAlchemy into the sync worker process. Under high concurrency (>8 simultaneous tasks) the thread pool exhausts, tasks stall for >1 hour, and `task_guardian_patrol` marks them as `ERROR`.

**Root cause B:** The Inference Server (`:8001`) is down or unhealthy.

**Diagnosis:**
```bash
# Check worker logs
docker compose logs worker --tail=100

# Verify inference server health
curl http://localhost:8001/health

# Inspect zombie tasks marked by the Guardian
docker compose exec postgres psql -U aicopilot -c \
  "SELECT id, status, current_step, updated_at FROM \"AgentTask\" \
   WHERE status='ERROR' ORDER BY updated_at DESC LIMIT 10;"
```
